Admin + User Panel — App Structure

Should these be one combined app or two separate deployments?

A

One Combined App

Single deployment. Role-based routing — customers see the user dashboard, staff see the admin dashboard. One codebase, one URL.

  • Simpler to deploy and maintain
  • Shared auth, shared components
  • Admin views gated by role, not by URL
B

Two Separate Apps

User panel at one URL (e.g. panel.cobalt.gg), admin panel at another (e.g. admin.cobalt.gg). Independent deployments.

  • Hard separation — admin app never ships to users
  • Two codebases to maintain
  • Admin app can live on localhost / VPN only
